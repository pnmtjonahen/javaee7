JavaEE7 JAX RS with DIGEST Authentication - Additional Information

This is a implementation of the person application as it described in "RESTFul Java WebServices Security" with additional Glasshfish configuration

  • Create database, use the create.sql to create the database, adn insert some users (see also the .sql scripts)
  • Create connectionPool and database resource, do not add the username/password to the connection pool, this prevents others to use the pool to read user info
  • Create the jdbcDigestReal securityrealm, for Group table name do not use the groups table but use the users_groups join table
